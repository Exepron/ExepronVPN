; BIND zone file for the Exepron Private DNS zone (exepron.com).
;
; Contains ALL records VPN-connected clients need to resolve:
;   - Internal-only services (sales, admin, salestest, admintest) -> private IPs in 172.16.0.0/16
;   - Public services mirrored from the public exepron.com zone -> their public IPs
;
; Why mirror the public records here:
;   When a Private DNS Zone is linked to a VNet, it is authoritative for
;   *.exepron.com queries from VNet-attached resources (which includes VPN
;   clients via the tunnel). Missing records return NXDOMAIN; there is no
;   fallback to public DNS. So every public name VPN users need to reach must
;   also exist here, with the SAME public IP. Traffic to public IPs still flows
;   over the regular internet because only 172.16.0.0/16 is routed through the
;   VPN tunnel.
;
; Import command (from Azure Cloud Shell or local az CLI):
;   az network private-dns zone import -g Exepron -n exepron.com -f private-zone-records.txt
;
; This is an upsert: same-name same-type records are replaced, others are
; left untouched. Auto-managed records (SOA, NS) are not in this file and
; remain unchanged.
